# Agent 当前回合上下文包

## 一句话理解

模型每一轮真正看到的不是“用户刚说的一句话”，而是 Agent Runtime 临时拼出来的上下文包。

## 上下文包里可能有什么

| 内容 | 作用 |
| --- | --- |
| System / developer rules | 定义角色、安全边界、工具规则、输出格式。 |
| 项目规则 | 例如 `AGENTS.md`、`TOOLS.md`、仓库规范。 |
| 用户输入 | 当前任务、问题、补充信息。 |
| 对话历史或摘要 | 保持多轮连续性。 |
| 记忆检索结果 | 把长期记忆里相关片段带回本轮。 |
| 文件内容 | 当前任务需要看的代码、文档或资料。 |
| 工具定义 | 告诉模型有哪些工具以及参数 schema。 |
| 工具执行结果 | 上一轮工具调用后的结果。 |
| 当前环境和任务状态 | 工作目录、时间、运行状态、待办、限制条件。 |

## 核心边界

上下文包是“本轮模型实际可见的信息”。记忆、文件、网页和数据库可以存在，但如果没有被检索、读取、整理并注入上下文，模型本轮就不能直接使用它们。

```text
外部资料存在
不等于
模型本轮看到了这些资料
```

## 为什么这很重要

很多 Agent 行为都可以从上下文包解释：

| 现象 | 解释 |
| --- | --- |
| 模型好像记得你之前说过的话 | 历史、摘要或记忆检索被带入了上下文。 |
| 模型忘了之前细节 | 历史没有带入，或摘要丢了细节。 |
| 模型知道项目规则 | `AGENTS.md` 或类似规则被注入。 |
| 模型会调用工具 | 工具 schema 在上下文或 API 参数里可见。 |
| 模型能基于文件回答 | 文件内容被读取后放进上下文。 |

## 常见误区

| 误区 | 更准确的理解 |
| --- | --- |
| Agent 每次只把用户消息发给模型 | 通常还会带规则、历史、记忆、文件、工具等。 |
| 有记忆就一定会用到 | 只有检索并注入当前上下文的记忆才会被使用。 |
| System Prompt 等于完整上下文 | System Prompt 只是上下文包的一部分。 |
