# 上下文拼接不等于模型记忆

## 一句话理解

AI 产品里“记住了你说过的话”，多数情况下不是模型权重被更新，而是历史、摘要或外部记忆被重新放进了本轮上下文。

## 严格区分

| 层 | 含义 | 是否等于模型权重变化 |
| --- | --- | --- |
| 上下文窗口 | 本轮请求里模型能看到的文本、工具定义和结果。 | 否 |
| 对话历史 | 过去消息被原样或摘要带入。 | 否 |
| 外部长期记忆 | 存在文件、数据库、向量库里的信息。 | 否 |
| 项目文件 | Agent 读取到的本地仓库或文档。 | 否 |
| 模型参数 | 训练后固化在模型权重里的知识。 | 是，但普通对话通常不会即时改参数 |

## 产品体验和机制的差异

产品体验上，用户会觉得“它记得我”。机制上，更常见的是：

```text
历史或记忆被保存
-> 本轮检索或摘要
-> 注入上下文
-> 模型基于上下文回答
```

所以“记住”在体验上成立，但从模型机制看，更像“运行时把相关信息重新提供给模型”。

## 例子

如果 Agent 知道你偏好短、清楚、适合手机阅读的总结，可能有三种原因：

1. 这条偏好还在当前对话历史里。
2. 这条偏好被写进了长期记忆，并在本轮被检索出来。
3. 这条偏好被写进项目规则或用户配置，并在启动时注入上下文。

这三种都不是模型参数即时改变。

## 常见误区

| 误区 | 更准确的理解 |
| --- | --- |
| AI 记住了 = 模型训练了 | 普通使用中通常是外部记忆或历史被重新注入。 |
| 记忆存在 = 模型一定知道 | 只有进入当前上下文才会被本轮使用。 |
| 上下文越多越好 | 太多上下文会增加噪声、成本和冲突。 |

## 自测问题

1. 为什么模型会“忘记”之前说过的内容？
2. 外部记忆和当前上下文有什么区别？
3. 什么情况下才需要把信息写入长期记忆？
