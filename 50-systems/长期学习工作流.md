# 长期学习工作流

## 设计目标

这套流程服务于长期学习，不服务于短期记得更多。

判断标准：

1. 能从记忆中说出来。
2. 能用自己的例子解释。
3. 能写代码、推导或实验验证。
4. 能和已有项目经验连接。
5. 能在几周后重新找回并继续推进。

## 核心循环

```text
Capture -> Understand -> Retrieve -> Build -> Teach -> Review
```

### 1. Capture

学习时不要边听边整理系统。先保留注意力。

放到：

```text
00-inbox/YYYY-MM-DD-主题-随手记录.md
```

只记录四类内容：

- 关键词
- 老师/讲者的关键判断
- 自己没听懂的问题
- 值得之后验证的想法

### 2. Understand

当天或第二天，把原始记录整理成来源笔记。

放到：

```text
10-sources/courses/课程名/YYYY-MM-DD-课时标题.md
10-sources/lectures/YYYY-MM-DD-讲座标题.md
10-sources/papers/论文标题.md
10-sources/books/书名-章节.md
```

来源笔记只回答：

- 这个材料在讲什么。
- 它的论证链路是什么。
- 我有哪些问题。
- 哪些概念值得提炼。

### 3. Retrieve

整理来源笔记后，立刻做一次主动回忆。

规则：

- 合上课程资料。
- 不复制原文。
- 用自己的话写 5-10 个自测问题。
- 每个问题先凭记忆回答，再回看资料修正。

主动回忆写在来源笔记末尾，或拆到 `20-concepts/`。

### 4. Build

技术学习必须经过验证，否则容易停留在“听懂了”。

验证方式按优先级：

1. 从零实现一个最小版本。
2. 复现一个实验或图。
3. 改一个变量观察结果。
4. 把概念接入真实项目。
5. 写一段反例说明它什么时候失效。

代码放到：

```text
40-code-labs/主题名/
```

项目经验放到：

```text
30-projects/项目名/
```

### 5. Teach

能讲清楚才算真正进入自己的知识系统。

输出可以很小：

- 一张概念卡片
- 一段 QQ 分享
- 一篇短文
- 一个 README
- 一个图解

概念卡片放到：

```text
20-concepts/概念名.md
```

每张概念卡片至少包含：

- 一句话理解
- 核心机制
- 例子
- 常见误区
- 和其他概念的关系
- 我的判断

### 6. Review

每周固定复盘一次，不靠临时兴致。

放到：

```text
90-reviews/YYYY-Www.md
```

复盘只问五个问题：

1. 本周真正掌握了什么。
2. 哪些只是看过或听过。
3. 哪些概念需要代码验证。
4. 哪些问题下周继续追。
5. 哪些笔记值得沉淀进 `50-systems/`。

## 每种内容的去向

| 内容 | 位置 | 完成标准 |
| --- | --- | --- |
| 随手记录 | `00-inbox/` | 不丢失，不打断学习 |
| 课程/讲座/论文笔记 | `10-sources/` | 保留来源、论证和问题 |
| 概念理解 | `20-concepts/` | 能脱离原材料讲清楚 |
| 真实项目 | `30-projects/` | 有背景、决策、结果、复盘 |
| 代码实验 | `40-code-labs/` | 能运行，有结论 |
| 方法论 | `50-systems/` | 可复用，可指导下一次行动 |
| 周/月复盘 | `90-reviews/` | 明确下一步 |

## 质量门槛

一条笔记是否值得长期保留，看它能否通过至少一个检查：

- 它能回答一个以后会重复遇到的问题。
- 它能减少下一次实践的试错成本。
- 它能解释一个关键概念。
- 它能连接两个原本分散的主题。
- 它能作为代码实验或项目决策的依据。

## 不推荐的做法

- 大量高亮但不提问题。
- 只整理目录，不做主动回忆。
- 只收藏链接，不记录自己为什么收藏。
- 概念卡片只是搬运原文。
- 代码实验没有 README 和结论。
- 复盘只写做了什么，不写判断变化。

## 推荐节奏

### 每次学习后 10 分钟

- 把 `00-inbox/` 的当天记录补完整。
- 写下 3 个问题。
- 标出 1-3 个可提炼概念。

### 每周 60-90 分钟

- 清空或处理 `00-inbox/`。
- 提炼 2-5 张概念卡片。
- 做一次主动回忆。
- 规划一个代码实验或项目验证。
- 写周复盘。

### 每月 2 小时

- 合并重复概念。
- 更新学习地图。
- 把稳定判断移入 `50-systems/`。
- 删除低价值材料。
