## 随手记录

这门课的主线更接近“现代 LLM / AI Agent 系统课”，不是从传统监督学习基础开始。

课程内容大致分为：

- AI Agent：课程规则、OpenClaw 例子、Context Engineering、Agent 之间的互动、Agent 对工作的影响。
- 模型内部与推理效率：Flash Attention、KV Cache、Positional Embedding、长输入处理。
- 如何教育模型：Harness Engineering、Self-Correction。
- 模型自我成长：Self-Improving。
- 模型训练与改造作业：Training Transformer、Finetuning without Forgetting、Model Editing、Model Merging、Test-Time Scaling、Flow Matching。
- 语音语言模型：Spoken Language Model。

## Hermes / OpenClaw 初步理解

- Hermes 更像“agent runtime + 长期记忆/技能沉淀系统”：同一个 `AIAgent` 核心被 CLI、Gateway、Cron、ACP、API 等触发入口复用；重点是 prompt 组装、provider 解析、工具调度、SQLite 会话检索、profile 隔离、skills/memory/curator 等能力。
- OpenClaw 更像“gateway-first 的个人 agent 编排系统”：一个长期运行的 Gateway 拥有聊天渠道、控制端、节点设备、消息路由、agent turn、channel、workspace、skills、plugins 和安全配对。
- 暂时不要把两者理解成“谁更高级”。更好的学习框架是：OpenClaw 帮助理解 agent 如何接入现实世界的多个聊天渠道；Hermes 帮助理解 agent 如何在长期使用中记忆、复用技能、形成个人化行为。
- 初步使用判断：写代码、改仓库、跑测试，优先用 Codex 或 Claude Code；需要长期在线、跨聊天渠道、定时自动化、记住个人偏好并沉淀技能时，才考虑 Hermes。
- 修正：Codex 也能完成很多“学习助理、项目秘书、定时检查、笔记整理、技能化工作流”的事情。Hermes 的差异不一定是能力独有，而是把这些能力做成一个长期在线、profile 隔离、gateway/cron/memory/skills 内建的自托管运行时。
- 概念问题：如何理解“自托管”和“profile”？自托管是运行位置和控制权问题：agent 跑在自己的电脑、VPS 或容器里，数据、密钥、进程、入口和更新由自己负责。profile 是状态隔离问题：同一个 Hermes 程序可以创建多个互不混用的 agent 身份，每个 profile 有自己的配置、记忆、会话、技能、定时任务和 gateway 状态。
- 试用建议：不要先用 Hermes 替代 Codex 写代码。优先试三个能体现 Hermes 差异的功能：为 `agent_learning` 创建独立 learning profile；创建一个只读学习复盘 cron；再试一次 QQBot/gateway 渠道，把同一个 agent 从聊天渠道唤起。

## 值得验证的想法

- 做一个最小 agent loop，对比：无检索、带检索、带工具、带子 agent 压缩。
- 做一个 KV Cache / attention 可视化小实验，帮助理解推理加速。
- 用一个小模型或模拟任务复现 model editing / model merging 的核心现象。
- 把课程里的作业主题映射到自己的项目经验：QQ bot、Codex 使用、量化空间。
